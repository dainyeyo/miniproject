# 제품 요구사항 정의서 (PRD) - 대기실 강퇴, 준비 및 중도 입장 차단 시스템

## 1. 개요
본 문서는 EGGG(AI 드로잉 퀴즈 게임)의 대기실 내 방장 권한 강퇴 기능(우측 상단 absolute X 버튼 스타일), 대기실 준비 시스템 및 게임 시작 후 중도 참여 차단 시스템 사양을 명세합니다.

## 2. 주요 기능 및 요구사항 (User Stories)

### 2.1 대기실 강퇴 기능 (Host Eviction)
1. **우측 상단 Absolute 'X' 강퇴 버튼 UI**
   - 플레이어 카드(`.lobby-player-slot`)에 `position: relative;`를 부여합니다.
   - 강퇴 버튼(`.kick-player-btn-absolute`)은 flex layout 흐름에서 제외되도록 `position: absolute;`를 활용하여 플레이어 카드의 우측 상단(`top: 6px; right: 8px;`)에 작게 '×' 마크로 absolute 정렬합니다.
   - 이를 통해 닉네임의 길이 및 다른 인라인 요소(준비완료/방장 뱃지 등)의 너비와 겹치거나 깨지는 현상을 원천 방어합니다.
   - 방장 본인의 플레이어 카드에는 노출하지 않습니다.
   - 버튼은 빨간색 원형 뱃지 스타일을 차용하여 작지만 시인성이 높고 카툰풍 테마에 부합하도록 에스테틱을 설계합니다.
2. **강퇴 트랜잭션**
   - `/api/rooms/action` API (`action: 'kick-player'`)를 통해 방장 검증을 통과한 후 봇은 물리 삭제(`DELETE`), 유저는 상태값 변경(`status = 'kicked'`, `is_active = FALSE`) 및 시스템 퇴장 공지.
3. **폴링 동기화 및 튕김 알림**
   - 폴링 API 호출 중 본인이 강퇴되었음이 감지되면 즉시 플레이어 레코드 파괴 후 403 `kicked: true` 리턴하여 클라이언트 퇴장 조치.

### 2.2 대기실 준비 시스템 (Room Ready System)
- 일반 플레이어는 `준비완료` (초록색) 또는 `준비중` (연회색) 뱃지 노출.
- 게스트 하단 버튼 바에 `"준비 완료"` / `"준비 취소"` 버튼 제공하여 실시간 상태 토글 (`action: 'toggle-ready'`).
- 신규 조인 시 초기 `status = 'waiting'` 설정, 게임 복귀 시 게스트 상태 `waiting` 리셋.
- 방장 제외 전체 참가자가 준비완료(`ready`) 상태일 때만 방장의 "게임 시작 (Start)" 버튼 활성화.

### 2.3 게임 중 중도 입장 차단 시스템 (In-Game Join Prevention)
- 방의 상태가 대기실(`waiting`) 상태일 때만 입장을 수락합니다.
- 대기실이 아닌 경우, 입장을 전면 거부하고 HTTP status `400`과 함께 `"이미 게임이 진행 중이거나 종료된 방입니다."` 오류 응답을 반환합니다.

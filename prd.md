# 제품 요구사항 정의서 (PRD) - 대기실 강퇴, 준비 및 중도 입장 차단 시스템

## 1. 개요
본 문서는 EGGG(AI 드로잉 퀴즈 게임)의 게임 대기실에서의 방장 권한 강퇴 기능, 참여자 준비 상태 시스템, 그리고 게임이 시작된 이후 외부 게스트가 대기방 코드를 통해 임의로 중도 참여하는 행위를 방지하는 입장 제약 사양을 정의합니다.

## 2. 주요 기능 및 요구사항 (User Stories)

### 2.1 대기실 강퇴 기능 (Host Eviction)
- 방장(Host) 권한의 플레이어 카드 우측 영역에 타인의 카드를 강퇴할 수 있는 `.kick-player-btn` 버튼 노출.
- `/api/rooms/action` API (`action: 'kick-player'`)를 통해 봇은 물리 삭제(`DELETE`), 유저는 상태값 변경(`status = 'kicked'`, `is_active = FALSE`) 및 시스템 퇴장 공지.
- 폴링 동기화 중 `status === 'kicked'`가 감지되면 즉시 플레이어 레코드 파괴 후 403 `kicked: true` 리턴하여 클라이언트 퇴장 조치.

### 2.2 대기실 준비 시스템 (Room Ready System)
- 일반 플레이어는 `준비완료` (초록색) 또는 `준비중` (연회색) 뱃지 노출.
- 게스트 하단 버튼 바에 `"준비 완료"` / `"준비 취소"` 버튼 제공하여 실시간 상태 토글 (`action: 'toggle-ready'`).
- 신규 조인 시 초기 `status = 'waiting'` 설정, 게임 복귀 시 게스트 상태 `waiting` 리셋.
- 방장 제외 전체 참가자가 준비완료(`ready`) 상태일 때만 방장의 "게임 시작 (Start)" 버튼 활성화.

### 2.3 게임 중 중도 입장 차단 시스템 (In-Game Join Prevention)
1. **입장 권한 상태 기반 제약**
   - 방에 초대 코드로 참여 시도 시(`join` API 호출), 해당 방의 상태(`status`)를 검증합니다.
   - 방의 상태가 **대기실(`waiting`) 상태일 때만** 입장을 수락합니다.
2. **비대기 상태의 차단 및 에러 전파**
   - 방의 상태가 게임 진행 중(`game`)이거나 결과 스크린(`result`) 등 대기실이 아닌 경우, 입장을 전면 거부하고 HTTP status `400`과 함께 `"이미 게임이 진행 중이거나 종료된 방입니다."` 라는 예외 에러 메시지를 반환합니다.
3. **클라이언트 피드백**
   - 프론트엔드는 400 에러 수신 시, 입력 필드 하단에 경고 문구를 빨간색 텍스트로 가독성 높게 표시하여 유저의 어뷰징 시도를 실시간 차단합니다.

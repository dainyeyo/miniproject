# 제품 요구사항 정의서 (PRD) - 대기실 강퇴, 준비, 중도 입장 차단, 차별화된 봇 이름 및 최고 득점자 방장 위임 시스템

## 1. 개요
본 문서는 EGGG(AI 드로잉 퀴즈 게임)의 대기실 내 방장 권한 강퇴 기능(모서리에 걸친 absolute X 버튼 스타일), 대기실 준비 시스템, 게임 시작 후 중도 참여 차단 시스템, 차별화된 봇 이름 시스템 및 게임 종료 시 최고 득점자 방장 자동 승격 시스템 사양을 명세합니다.

## 2. 주요 기능 및 요구사항 (User Stories)

### 2.1 대기실 강퇴 기능 (Host Eviction)
- 플레이어 카드(`.lobby-player-slot`)에 `position: relative;`를 부여하고, 강퇴 버튼(`.kick-player-btn-absolute`)은 `position: absolute; top: -10px; right: -10px;`로 모서리 경계선에 걸치도록 정렬합니다.

### 2.2 대기실 준비 시스템 (Room Ready System)
- 일반 플레이어는 `준비완료` (초록색) 또는 `준비중` (연회색) 뱃지 노출, 게스트는 `"준비 완료"` / `"준비 취소"` 실시간 토글. 방장 제외 전체 참가자가 준비완료(`ready`) 상태일 때만 방장의 "게임 시작 (Start)" 버튼 활성화.

### 2.3 게임 중 중도 입장 차단 시스템 (In-Game Join Prevention)
- 방의 상태가 대기실(`waiting`) 상태일 때만 입장을 수락하고, 아닌 경우 HTTP 400 Bad Request 에러를 반환합니다.

### 2.4 차별화된 봇 이름 및 식별 표시 시스템 (Differentiated Bot Names)
- 대기실에서 "봇 추가" 클릭 시, 초대되는 모든 봇의 닉네임 뒤에 ` (봇)` 식별 태그를 자동으로 주입하고 중복 봇 조인을 방지합니다.

### 2.5 최종 라운드 종료 시 최고 득점자 방장 자동 위임 시스템 (High Scorer Host Promotion)
1. **위임 트리거 조건 (Promotion Trigger)**
   - 게임의 마지막 라운드에서 전원 정답 또는 타이머 만료 등으로 게임이 종료되는 시점(즉, 방의 `status`가 `'result'`로 업데이트되는 시점)에 기동합니다.
   - 방장이 중도에 수동으로 게임 종료(`game-over` 액션)를 선언하여 게임이 최종 종료되는 상황에서도 동일하게 트리거를 활성화합니다.
2. **방장 위임 대상 유저 선정 알고리즘 (Selection Algorithm)**
   - 봇은 방장 역할을 수행할 수 없으므로(게임 시작 조작 불가능), 봇이 아니며(ID가 `BOT-`로 시작하지 않음) 현재 접속 중인 활성 휴먼 유저(`is_active = TRUE`) 중 **가장 누적 점수(`score`)가 높은 1인**을 선정합니다.
   - 동점자가 발생할 경우 닉네임 사전순(`ORDER BY score DESC, nickname ASC`)으로 단 1인만을 결정론적으로 선정합니다.
3. **방장 권한 교체 트랜잭션**
   - 기존의 모든 유저의 방장 컬럼을 비활성화(`is_host = FALSE`)한 후, 선정된 최고 득점 유저의 방장 권한을 승격(`is_host = TRUE`)시킵니다.
   - 시스템 메시지로 대방에 `👑 최고 득점자인 [닉네임]님이 새로운 방장이 되었습니다!` 공지를 자동으로 추가하여 상태 변화를 알립니다.
   - 대기실 복귀(`go-lobby`) 시, 새로운 방장은 대기방 상태가 `'ready'`로, 타 게스트 유저들은 `'waiting'`으로 정상 연동되도록 보장합니다.

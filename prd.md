# 제품 요구사항 정의서 (PRD) - 대기실 강퇴 및 준비 시스템

## 1. 개요
본 문서는 EGGG(AI 드로잉 퀴즈 게임) 대기실에서의 방장 권한 강퇴 기능과 더불어, 방장을 제외한 플레이어들의 준비(Ready) 상태 전제 하에 게임 시작을 강제하는 준비 시스템 사양을 명세합니다.

## 2. 주요 기능 및 요구사항 (User Stories)

### 2.1 대기실 강퇴 기능 (Host Eviction)
1. **방장용 강퇴 버튼 UI**
   - 방장(Host) 권한의 플레이어 카드 우측 영역에 타인의 카드를 강퇴할 수 있는 `.kick-player-btn` 카툰 스타일 버튼 노출.
   - 방장 본인 카드에는 노출하지 않음.
2. **강퇴 트랜잭션**
   - `/api/rooms/action` API (`action: 'kick-player'`)를 통해 방장 검증을 통과한 후 봇은 물리 삭제(`DELETE`), 유저는 상태값 변경(`status = 'kicked'`, `is_active = FALSE`) 및 시스템 퇴장 공지 채팅 등록.
3. **폴링 동기화 및 튕김 알림**
   - `/api/rooms/status` 호출 시 `status === 'kicked'`가 감지되면 즉시 플레이어 레코드 파괴 후 403 `kicked: true` 리턴.
   - 클라이언트에서 403 응답 수신 시 폴링을 끄고 `"강퇴 당하였습니다"` alert와 함께 랜딩 스크린으로 퇴장.

### 2.2 대기실 준비 시스템 (Room Ready System)
1. **준비 상태 뱃지 노출**
   - 대기실의 각 플레이어 슬롯에 상태 표시 뱃지(`.slot-badge.ready-badge`)를 시각적으로 노출합니다.
   - 방장은 `방장` (노란색/주황색), 일반 플레이어는 `준비완료` (초록색) 또는 `준비중` (연회색)으로 분기하여 에스테틱을 제공합니다.
2. **준비 상태 토글 기능**
   - 게스트 플레이어의 하단 제어 바에 `"준비 완료"` / `"준비 취소"` 버튼을 추가하여 자신의 준비 상태를 실시간으로 변경(토글)할 수 있게 합니다.
   - `/api/rooms/action` API (`action: 'toggle-ready'`)를 호출하여 DB `players` 테이블의 `status`를 `'ready'`와 `'waiting'`으로 교차 업데이트합니다.
3. **입장 및 복귀 시 초기화**
   - 플레이어가 초대 코드로 참여할 때(`join` API), 초기 `status`를 `'waiting'`으로 설정하여 즉시 준비완료되지 않도록 처리합니다.
   - 게임 결과 화면에서 대기실로 복귀할 때(`action: 'go-lobby'`), 방장의 `status`는 `'ready'`, 그 외 게스트의 `status`는 `'waiting'`으로 다르게 세팅합니다.
4. **게임 시작 자격 조건 강제**
   - 방장을 제외한 모든 활성 플레이어(실제 게스트 + 봇)가 `'ready'` 상태일 때만 방장의 "게임 시작 (Start)" 버튼이 활성화됩니다.
   - 방장 외 게스트가 0명인 경우(단독 디버깅/테스트 등)에는 즉시 시작이 가능합니다.
   - 준비 완료 조건을 만족하지 못해 비활성화된 상태일 때, 방장 화면에 `"⚠️ 모든 참가자가 준비를 완료해야 시작할 수 있습니다."` 라는 에러 가이드라인을 출력합니다.
